Configuring Volume Migration

The objective of Storae Volume migration is to optimize QoS. Storae Volume migration helps to control the QoS parameters within a Pool and to move data. You can migrate Storae Volume from a Pool to within the same Node or to a different Node. Also, you can migrate Storae Volume across Sites.

To migrate Storage Volume,

1. In the Elasticenter Navigation pane Select Storage volumes

2. In the Actions Icon Click Schedule

3. In the following page,specify the required details and then click next.

4. Select the Pool with sufficient IOPS to migrate Storage Volume and then click Next.

5. Select the Backup interface, IP Address, and schedule of migration

6. Click ADD. Scheduling progress will start.

7. Once scheduling progress done. Click View Transfer Status in Action Icon and check whether the Status is Uptodate.

Note : check the Status is Uptodate or Not.once the status is Uptodate we can start the activation

8. To activate Volume migration select Migrate in Actions Icon. In the following Confirmation Dialog box Click Yes.

9. Wait for the successfully activate VSM messsage.
